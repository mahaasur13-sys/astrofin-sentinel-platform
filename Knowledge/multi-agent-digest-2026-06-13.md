## 1. Arbor — Microsoft Research: мультиагентный фреймворк с Hypothesis-Tree Refinement

- **Источник:** arXiv + Microsoft Research / GitHub (RUC-NLPIR/Arbor)
- **Краткое описание:** Arbor — это generalist autonomous research агент, который превращает долгосрочное исследование в кумулятивное обучение через персистентное дерево гипотез. Каждый эксперимент становится узлом с гипотезой, оценкой, артефактом и дистиллированным инсайтом; агенты ведут портфельный поиск по конкурирующим идеям вместо одной хрупкой траектории. На бенчмарках Arbor обгоняет Codex и Claude Code на 6 исследовательских задачах и достигает 86% Any-Medal на MLE-Bench Lite; параллельные публикации сообщают о 193% улучшении throughput-latency в LLM-инференсе при использовании tree-search как shared agent memory.
- **Применение для AstroFinSentinelV5:** Структура дерева гипотез отлично ложится на финансовый конвейер «аналитик → исследователь → риск-менеджер → трейдер»: каждый торговый сигнал можно оформить как узел дерева (гипотеза о движении, бэктест-скор, артефакт-график, инсайт), а рой параллельных solver-веток даст естественный pass@N-отбор лучших стратегий. Внедрение HTR вместо плоского debate-протокола должно сильно поднять стабильность оркестрации и прозрачность решений для ревью.

## 2. MARS — Multi-Agent Review System: дебаты с -50% токенов

- **Источник:** OpenReview (ACL ARR 2026)
- **Краткое описание:** MARS переосмысляет multi-agent debate (MAD) в виде ролевого ревью-процесса: один автор генерирует первичное решение, независимые ревьюеры выносят вердикты и комментарии, а мета-ревьюер синтезирует фидбэк и инициирует ревизии. За счёт отсутствия дорогих «ревьюер-ревьюер» взаимодействий MARS достигает сравнимого качества с MAD, тратя примерно вдвое меньше токенов и обеспечивая лучшую масштабируемость.
- **Применение для AstroFinSentinelV5:** Ролевая «ревью»-схема идеально подходит под сценарий «аналитик предложил сигнал → независимые ревьюеры-пруфридеры (риск, комплаенс, макро) → мета-агент выносит финальное решение»: это даст существенную экономию бюджета LLM на каждом торговом цикле и уменьшит «sycophantic drift», который часто встречается в плоских multi-agent конфигурациях.

## 3. DeLM — Decentralized Language Models: параллельные solver-ветки с общим контекстом

- **Источник:** GitHub (yuzhenmao/DeLM) + arXiv:2606.10662
- **Краткое описание:** DeLM заменяет централизованный оркестратор общей верифицированной памятью и очередью задач: несколько solver-тредов (PlannerAgent + SWEBenchImplementerAgent + Docker + локальная память) работают параллельно, обмениваясь компактными SharedLessons, а финальный pass@N берёт лучший результат среди параллельных прогонов. Архитектура показывает высокую эффективность на SWE-bench-like задачах и подчёркивает разделение Local Memory (потоковые наблюдения) и Shared Lessons (кросс-поточные факты).
- **Применение для AstroFinSentinelV5:** Паттерн «decoupled solvers + shared verified context» напрямую переносится на мультиагентную бэктест-лабораторию: десятки solver-агентов тестируют разные торговые гипотезы параллельно, разделяя уроки о рынке через SharedLessons (например, «в боковике RSI-дивергенция работает плохо»), а локально хранят свои экспериментальные артефакты. Это естественное усиление над текущим «центральным супервизором» в V5 — выходит устойчивее, дешевле и проще наблюдаемо.
