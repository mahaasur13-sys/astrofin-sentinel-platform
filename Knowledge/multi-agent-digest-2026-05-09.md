### 1. **Graph-of-Agents (GoA) — масштабируемый graph-based фреймворк для мультиагентной коллаборации LLM**

- **Источник:** arXiv (arXiv:2604.17148)
- **Краткое описание:** Фреймворк GoA представляет агентов как узлы направленного графа с model cards (описаниями доменной экспертизы). Вместо хаотичного подключения всех доступных агентов GoA выбирает только релевантных через семплирование по model cards, затем формирует рёбра на основе кросс-агентного оценивания ответов и запускает directional staged message passing — информация течёт от более релевантных агентов к менее, после чего обратно для финальной ревизии. Итоговый ответ формируется через graph-based pooling (max/mean). На бенчмарках MMLU, MMLU-Pro, GPQA, MATH, HumanEval, MedMCQA фреймворк показывает превосходство над подходами, использующими всех агентов, при этом задействуя лишь 3 из 6. Снижает token usage и повышает координированность при масштабировании числа моделей.
- **Применение для AstroFinSentinelV5:** Архитектура GoA идеально ложится на задачу маршрутизации финансовых данных между специализированными агентами (аналитик, риск-менеджер, портфельный менеджер). Model cards аналогично могут описывать домены каждого агента в AstroFinSentinelV5, а graph-based aggregation — использоваться для финального синтеза решений из нескольких агентов.

### 2. **Agent Q-Mix — RL-подход к динамическому выбору топологии коммуникации агентов**

- **Источник:** arXiv (arXiv:2604.00344)
- **Краткое описание:** Agent Q-Mix решает задачу выбора оптимальной коммуникационной топологии (кто с кем говорит) в мультиагентных LLM системах через reinforcement learning с QMIX-style value factorization. Каждый агент выбирает коммуникационные действия раундами, формируя динамический граф связей. Архитектура включает topology-aware GNN encoder + GRU memory + per-agent Q-heads, обучается в режиме CTDE (Centralized Training, Decentralized Execution). На 7 бенчмарках (coding, reasoning, mathematics) Agent Q-Mix показывает более высокую accuracy при улучшенной token efficiency и fault tolerance. Добился 20.8% на Humanity's Last Exam (HLE) с Gemini-3.1-Flash-Lite. Предложен механизм динамической топологии вместо статичных отф的定义.
- **Применение для AstroFinSentinelV5:** Динамическая топология коммуникации между агентами позволяет системе адаптивно перестраивать связи в зависимости от рыночных условий. Например, при высокой волатильности агент риска получает больше связей с аналитическими агентами, а в стабильных условиях — включается режим экономии токенов через менее связанную топологию.

### 3. **ComposioHQ/agent-orchestrator — оркестратор параллельных AI-агентов для автоматизации CI/CD и code review**

- **Источник:** GitHub (ComposioHQ/agent-orchestrator)
- **Краткое описание:** TypeScript-фреймворк (MIT-лицензия, @composio/ao@0.2.2) для оркестрации множества AI-агентов в параллели, каждый в собственном git worktree с отдельной веткой и PR. Система автономно обрабатывает CI failures, review comments, code reviews, минимизируя ручное вмешательство. Агенты работают через tmux, ConPTY/process или Docker, поддерживают Claude/Code модели. Единый dashboard для мониторинга всей активности. Ключевое отличие — agent-agnostic и runtime-agnostic архитектура: можно подключать любые модели и рантаймы без изменения системы.
- **Применение для AstroFinSentinelV5:** Паттерн параллельных агентов с независимыми рабочими пространствами хорошо масштабируется на задачи backtesting и исследования рынков. Агенты могут одновременно анализировать разные классы активов (акции, облигации, деривативы) в изолированных "воркtrees", а оркестратор агрегирует результаты через общий dashboard.
