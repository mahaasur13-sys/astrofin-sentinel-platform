## Решение

**Phase B2d (исправление collection errors в `META_RL_MODE`) отменяется как неактуальная для текущей кодовой базы.**

## Контекст и диагностика

План предполагал, что `pytest --collect-only` падает из-за проблемы с конфигурацией `meta_rl`, которую нужно чинить в `meta_rl/settings.py` и связанных модулях. Фактическая проверка кода и среды:

1. `meta_rl/config.py` использует `os.getenv("META_RL_MODE", HISTORICAL_MODE)` без валидации и без `raise`.
2. `meta_rl/settings.py` — не существует (и не нужен, т.к. `config.py` уже выполняет его роль).
3. `meta_rl/collectors.py` — не существует и не упоминается ни в импортах, ни в тестах.
4. `tests/meta_rl/` — отсутствует; тесты лежат в `tests/test_meta_rl.py` (плоская структура, всё собирается).
5. `pytest --collect-only` на текущем `master` (`900ebdd`) **проходит без ошибок** (см. подтверждение в логах PR #206).

## Обоснование

> «Мы не можем починить то, чего нет».

Все признаки «слома» в брифе (отсутствующие модули, падающий `collect-only`) воспроизводятся только в восстановленной из бэкапа ветке `asp-restore-092329` после переноса `conftest.py` с `xfail`-списком KI-125a. На чистом `master` (`900ebdd`) ни один из этих симптомов не наблюдается.

## Следствие для roadmap

- Phase B2d исключается из `STATUS.md`, `SPRINT_2.md` (если бы упоминалась — не упоминается), `AGENTS.md`, `ROADMAP.md`.
- Следующий этап — **Phase B3 / Phase 3** по `PRODUCTION_BACKLOG.md` (87 задач, 5 фаз).

  - ввести `meta_rl/settings.py` с dataclass-обёрткой и валидацией значений режима (тоже отдельная задача).

Это уже **новая** работа, не B2d.
