## 1. Arbor — структурированный tree search как cognition layer для автономных агентов

- **Источник:** arXiv (2606.12563), AMD AI Research
- **Описание:** Arbor — мульти-агентный фреймворк, в котором явное дерево скорированных гипотез выступает общей рабочей памятью для группы агентов. Архитектура «checks-and-balances»: Orchestrator делегирует задачи Domain Specialists, а Critic следит за стабильностью через root-cause анализ и валидацию измерений. Возможности агентов разделены на hard skills (доменная экспертиза) и soft skills (протоколы координации). На задаче full-stack оптимизации LLM-инференса фреймворк показал +193% throughput-latency относительно вендорских бейзлайнов.
- **Применение для AstroFinSentinelV5:** Взять паттерн явного дерева гипотез как shared working memory для связки аналитик-агент + critic-агент. Полезно для многошаговых финансовых расследований, где нужно сохранять контекст между итерациями и не терять «диагностический сигнал» от неудачных гипотез. Подходит для backtest-оптимизации стратегий через параллельные ветки проверок.

## 2. DeLM — децентрализованный multi-agent фреймворк с общим контекстом

- **Источник:** arXiv (2606.10662), проект yuzhenmao.github.io/DeLM/
- **Описание:** DeLM (Decentralized Language Models) координирует агентов без центрального контроллера: агенты берут задачи из общей очереди и записывают обратно компактные верифицированные результаты. На SWE-bench Verified показывает 65.7% с Gemini 3-Flash — примерно на 10 п.п. выше лучших централизованных подходов, при стоимости около $0.12 за задачу. Прогресс виден всем воркерам без главного merge-агента.
- **Применение для AstroFinSentinelV5:** Архитектура без «единого мозга» хорошо ложится на сценарий, когда несколько специализированных агентов (новости, цены, риск-метрики) параллельно пишут в общий контекстный слой. Можно использовать как референс для low-cost пайплайна разметки и скоринга торговых сигналов.

## 3. Citadel — open-source orchestration layer для Claude Code и Codex

- **Источник:** GitHub (helrigle007/Citadel)
- **Описание:** Citadel добавляет «операционный слой» поверх Claude Code и OpenAI Codex для запуска multi-agent рабочих процессов. Ключевые механики: долговременная проектная память в `.planning/`, лёгкий `/do` роутинг, lifecycle-хуки (file protection, quality gates, telemetry), Fleet-режим с параллельными агентами в изолированных worktree, четырёхуровневая масштабируемость (Skill → Marshal → Archon → Fleet). Также встроены discovery relay, cost-telemetry и сквозной campaign-стейт между сессиями.
- **Применение для AstroFinSentinelV5:** Подходит как «операционная оболочка» для оркестрации Claude/Codex-агентов вокруг финансового workflow: маршрутизация запросов, защита чувствительных файлов с кошельками/ключами, телеметрия по расходам, и запуск параллельных агентов-исследователей в отдельных worktree для разных активов.
