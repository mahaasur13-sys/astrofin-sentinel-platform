## Системные компоненты

Это базовые проверки и настройки, которые выполняются в начале установки.

| Название | Что делает | Зачем нужно | Профиль |
|----------|------------|-------------|---------|
| **Pre-flight Checks** | Проверяет структуру, sudo, ОС, пользователя, диск, сеть | Убедиться, что система готова к установке | Все |
| **UFW Firewall** | Настраивает межсетевой экран (по умолчанию — запрет входящих) | Защита от внешних подключений | Все |
| **sysctl hardening** | Применяет настройки ядра Linux (защита от kptr, dmesg, BPF, сетевая защита) | Укрепление ядра против эксплоитов | Все |
| **Unattended Upgrades** | Включает автоматическую установку обновлений безопасности | Система обновляется сама без участия | Все |

## Как работает установка

```
pop-os-setup.sh
  ├── Загружает lib/ (logging, utils, installer, profiles)
  ├── Определяет профиль (workstation / ai-dev / full / cluster)
  └── Последовательно выполняет stages/
        stage01_preflight.sh   ← проверки готовности
        stage02_auto_repair.sh ← исправления ОС
        ...
        stage05_zsh.sh         ← Zsh + Oh My Zsh
        stage07_docker.sh      ← Docker + Compose
        stage09_cuda.sh        ← CUDA (ai-dev / full)
        stage10_hardening.sh   ← Firewall + fail2ban
        ...
```

Каждый stage-скрипт идемпотентен: если компонент уже установлен — установка пропускается с сообщением `already installed`.

## Критичность компонентов

| Критичность | Значение | Примеры |
|-------------|----------|---------|
| **REQUIRED** | Без этого система не будет работать корректно | Pre-flight, Zsh, Git |
| **OPTIONAL** | Улучшает опыт, но не критично | Portainer, Tailscale |
| **CONDITIONAL** | Устанавливается только при включённом флаге или наличии оборудования | CUDA, NVIDIA Drivers |
